## 📋 Распиновка (Pinout) STM32F412Rx

### 💾 1. SD-карта, SDIO:

| Пин | SDIO | SD | Назначение |
| :--- | :--- | :--- | :--- |
| **PC8** | SDIO_D0 | DAT0 (Pin 7) | Линия данных 0 |
| **PC9** | SDIO_D1 | DAT1 (Pin 8) | Линия данных 1 |
| **PC10** | SDIO_D2 | DAT2 (Pin 9) | Линия данных 2 |
| **PC11** | SDIO_D3 | CD/DAT3 (Pin 1) | Линия данных 3 |
| **PC12** | SDIO_CK | CLK (Pin 5) | Тактовая частота шины (Clock) |
| **PD2** | SDIO_CMD | CMD (Pin 2) | Командная линия управления (Command) |

### 🔊 2. I2S2 -> PCM5102:

| Пин | I2S2 | PCM5102 | Назначение |
| :--- | :--- | :--- | :--- |
| **PB12** | I2S2_WS | LRCK | Выбор канала (Левый / Правый) |
| **PB13** | I2S2_CK | BCK | Битовый клок (Синхронизация) |
| **PB15** | I2S2_SD | DIN | Последовательный поток аудиоданных |

### 📺 3. Дисплей 3.5", ST7796:

| Пин | SPI1 | ST7796 | Назначение |
| :--- | :--- | :--- | :--- |
| **PA5** | SPI1_SCK | SCK / CLK | Тактирование шины SPI |
| **PA7** | SPI1_MOSI | MOSI / SDA | Передача пикселей данных |
| **PB0** | LCD_CS | CS | Выбор чипа (Chip Select) |
| **PB2** | LCD_DC | DC / RS / A0 | Выбор: Данные или Команда |
| **PB1** | LCD_RST | RESET / RST | Аппаратный сброс контроллера экрана |
| **PA1** | LCD_BL | LED / BL | Подсветка экрана (Включение дисплея) |

### 🎛 4. Кнопки:

| Пин | Номер | Назначение |
| :--- | :--- | :--- |
| **PB7** | Кнопка 1 | Предыдущий трек (Prev) |
| **PB6** | Кнопка 2 | Воспроизведение / Пауза (Play/Pause) |
| **PB5** | Кнопка 3 | Следующий трек (Next) |
| **PB4** | Кнопка 4 | Стоп (Зарезервирована) |
| **PB3** | Кнопка 5 | Прыжок на следующий каталог (Folder) |

## 📂 Структура проекта

├── cmsis/                # Файлы ядра ARM Cortex-M4 и заголовочники STM32
├── fs/                   # Файловая система FatFs (ChaN) и низкоуровневый diskio
├── audio/                # Драйвер аудио-инициализации I2S и WAV-декодер
├── disp/                 # Драйвер LCD дисплея
├── u8g2x/                # Форк u8g2 - только шрифты!
├── shffplayer/           # Программный комплекс плеера
│   ├── shuffle_player.c  # Основной конечный автомат, логика Shuffle и отрисовка UI
│   ├── shuffle_player.h  # Прототипы функций, конфигурация интерфейса (координаты ROW_Y)
│   ├── dir_manager.c     # Динамический скан флешки, поиск WAV-папок
│   └── dir_manager.h     # Настройки корневой директории
└── src/                  # main.c и обработчик нажатий кнопок button.c

## 🛠 Сборка и прошивка

Для сборки необходим инструмент **GNU Arm Embedded Toolchain** (`arm-none-eabi-gcc`) и утилита `make`.

1. Очистить старые файлы сборки:
   ```bash
   make clean
   ```
2. Скомпилировать и загрузить проект в память микроконтроллера:
   ```bash
   make flash
   ```
